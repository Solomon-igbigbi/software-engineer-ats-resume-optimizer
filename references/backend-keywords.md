# Backend & Full-Stack Engineering Keywords Reference

Use this as a comprehensive reference when checking keyword coverage for backend or full-stack resumes. Not every resume needs all of these — the goal is to identify meaningful gaps given the candidate's experience level and target role.

---

## Role Terms
- Backend Engineer, Backend Developer, Software Engineer (Backend)
- Full Stack Engineer, Full Stack Developer, Software Engineer (Full Stack)
- API Developer, Platform Engineer, Systems Engineer
- Senior / Staff / Principal (level qualifiers)

## Architecture & Design Patterns
- REST API, RESTful API, RESTful architecture
- GraphQL, gRPC, Protocol Buffers (protobuf)
- Microservices architecture, monolithic architecture, service-oriented architecture (SOA)
- Event-driven architecture, event sourcing, CQRS
- Domain-driven design (DDD)
- Distributed systems, distributed architecture
- System design, high-level design
- Scalability, horizontal scaling, vertical scaling
- High availability, fault tolerance, resilience
- API gateway, service mesh, sidecar pattern
- Serverless architecture, function-as-a-service (FaaS)
- Webhook integration, webhook delivery

## Performance & Reliability
- Caching strategy, cache invalidation
- Redis, Memcached, in-memory caching
- CDN (Content Delivery Network)
- Performance optimization, latency reduction, throughput improvement
- p99 latency, response time, SLA, SLO
- Rate limiting, throttling
- Circuit breaker pattern, retry logic, exponential backoff
- Load balancing, auto-scaling
- Blue/green deployment, canary release
- Chaos engineering, resilience testing

## Observability & Operations
- Logging, structured logging, log aggregation
- Monitoring, alerting, dashboards
- Observability, distributed tracing, OpenTelemetry
- Metrics collection, Prometheus, Grafana
- APM (application performance monitoring), Datadog, New Relic
- Error tracking, Sentry
- On-call, incident response, postmortem
